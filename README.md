# Расчёт скидок дилеров

Веб-приложение для расчёта скидок дилеров КОМПО по шкалам выполнения условий работы ТПС 2026 финансового года. Приложение парсит XLSX-файл с условиями и шкалами, CSV-файл с фактическими отгрузками, динамически рассчитывает скидки по оборудованию, расходным материалам и сервису, и экспортирует результаты в Excel.

## Возможности

- **Сводная таблица выполнения условий** — KPI-карточки + таблица с планом/фактом/выполнением по каждому дилеру, динамический расчёт % скидки по оборудованию и расходным материалам на основе шкал. Скидка по РМ разделена на две колонки: «Скид. % (объём)» — за объём закупок (0–5%) и «Скид. % (мин. ост.)» — за наличие минимальных остатков (1% для РФ, «—» для Заруб).
- **Скидка по сервису** — отдельная таблица с накопительным фактом сервиса за 12 месяцев, расчёт % скидки и суммы в евро.
- **Детализация по дилеру** — помесячные отгрузки (12 месяцев финансового года, апрель→март) с квартальными и полугодовыми итогами.
- **Шкалы скидок** — все шкалы с подсветкой активного тира для выбранного дилера, отдельные StatCards для скидки по РМ (объём + мин. остатки).
- **Управление данными** — загрузка новых XLSX и CSV (multipart upload), синхронизация с локальным диском, **реальная синхронизация с Google Drive через service account**, ссылка на папку Drive с исходными файлами.
- **Экспорт в Excel** — выгрузка всех таблиц и итогов в .xlsx (4 листа: Сводная, Сервис, Данные, Шкалы).

## Бизнес-логика расчёта

Финансовый год начинается в **апреле** (Апр=месяц 1, Мар=месяц 12).

- **Российские дилеры (РФ)** — отчётность поквартальная (`/кв`):
  - Скидка по оборудованию: объём факта за квартал (в тыс. евро) → тир шкалы «Оборудование — за квартал» → строка «ИТОГО по МП».
  - Скидка по расходным материалам: объём факта за квартал (в тыс. евро) → тир шкалы «Расходные материалы» → сумма всех строк.
  - Скидка по сервису: объём факта за квартал по всем категориям / 2500 → тир шкалы «Оборудование» → строка «ИТОГО по МП» (даёт чистый %, напр. 14, 17, 19).
  - Сумма скидки сервиса в евро = накопленный факт × % / 100.

- **Зарубежные дилеры (Заруб)** — отчётность по полугодиям (`/пг`):
  - Скидка по оборудованию: объём факта за полугодие (в тыс. евро) → тир шкалы «Оборудование — за полугодие».
  - Скидка по расходным материалам: аналогично по шкале «Расходные материалы».
  - Скидка по сервису: накопленный факт за 12 месяцев (в абсолютных евро) → тир шкалы «Сервис (ЗЧ)» → строка «% скидки» (значения 0.1, 0.11, 0.13 — нормализуются умножением на 100).

Подробности — в [`src/lib/discount.ts`](src/lib/discount.ts) и [`src/lib/calc.ts`](src/lib/calc.ts).

## Быстрый старт

### Требования

- Node.js 20+ или Bun 1.3+
- (Опционально) Caddy для reverse proxy

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/<your-org>/dealer-discount-calculator.git
cd dealer-discount-calculator

# Установить зависимости
bun install
# или: npm install

# Скопировать .env.example → .env
cp .env.example .env

# Подготовить данные (опционально — для первого запуска)
mkdir -p data/uploads
# Положить файлы в data/uploads/:
#   - plans.xlsx — XLSX с условиями работы ТПС 2026 (вкладки по дилерам)
#   - facts.csv  — CSV с фактическими отгрузками (категория, дилер, дата, сумма)
```

### Формат входных файлов

**`plans.xlsx`** — каждая вкладка (кроме `Лист1`, `Лист2`, `Лист3`) представляет одного дилера:

| Строка | Содержимое |
|---|---|
| R2 (header) | `Выручка 2024 | Выручка 2025 | План 2026 | ...` |
| R3 | Имя дилера + плановые значения (годовые) |
| R4 | `Сервис` + план |
| R5 | `Оборудование` + план |
| R6 | `Расходные материалы` + план |
| R11+ | Шкалы скидок (Оборудование → РМ → Сервис (ЗЧ) → Система контроля) |

**`facts.csv`** — колонки: `Категория, Дилер, Дата, Сумма`. Категории: `Сервис`, `Оборудование`, `Расходные материалы`. Дата в формате `YYYY-MM-DD HH:MM:SS`.

### Запуск в режиме разработки

```bash
bun run dev
# или: npm run dev
```

Откройте http://localhost:3000.

### Production-сборка

```bash
bun run build
```

## Управление данными

| Действие | Где | Что делает |
|---|---|---|
| Загрузить новые XLSX/CSV | Вкладка «Данные» → «Загрузить XLSX/CSV» | POST multipart на `/api/upload`, сохраняет файлы в `data/uploads/`, парсит, обновляет состояние |
| Синхронизировать с Google Drive | Вкладка «Данные» → «Синхронизировать с Drive» | Через service account JSON ищет в папке Drive новейшие `Условия*.xlsx` и `Выполнение*.csv`, скачивает в `data/uploads/`, парсит |
| Синхронизировать с диском | Вкладка «Данные» → «Синхронизировать» | Перечитывает `data/uploads/` и перестраивает состояние |
| Открыть Google Drive | Вкладка «Данные» → «Открыть папку Drive» | Открывает исходную папку с файлами |
| Экспорт в Excel | Хедер → «Excel» | Скачивает `.xlsx` с 4 листами (Сводная, Сервис, Данные, Шкалы) |
| Сменить квартал | Хедер → селектор квартала | Меняет расчётный период для РФ; для Заруб — переключает 1/2 полугодие |

### Google Drive sync (опционально)

Для работы синхронизации с Google Drive:

1. Создать service account в Google Cloud Console
2. Включить Drive API
3. Скачать JSON-ключ и положить в `upload/` (имя файла должно содержать `metal-circle`, `service-account` или `sa-`)
4. Добавить email service account (`xxx@yyy.iam.gserviceaccount.com`) в качестве **Viewer** на папку Drive
5. Указать ID папки в `src/lib/drive-sync.ts` (`FOLDER_ID`)

См. [`src/lib/drive-sync.ts`](src/lib/drive-sync.ts) — использует `googleapis` с scope `drive.readonly`.

## Безопасное обновление production (safe-deploy)

`scripts/safe-deploy.sh` выполняет обновление production-сервера без потери данных:

1. Снапшотит живой `state.json` и `uploads/` обратно в исходники (чтобы пользовательские изменения пережили rebuild)
2. Останавливает `next-server`
3. Делает `bun run build`
4. Копирует `.next/static`, `public/`, `data/`, `upload/`, `db/`, `.env` в `.next/standalone/` (merge, не clobber)
5. Запускает сервер через `setsid -f` (переживает закрытие терминала)
6. Health-check через `curl`

```bash
bash scripts/safe-deploy.sh
```

Лог сервера: `/tmp/server.log`.

## Исключение дилеров

В [`src/lib/parse-plans.ts`](src/lib/parse-plans.ts) определён массив `EXCLUDED_DEALERS` — список подстрок (в нижнем регистре), по которым дилеры исключаются из приложения (совпадение проверяется по имени листа И по отображаемому имени).

```ts
// В XLSX есть ДВА листа с одинаковым отображаемым именем "Богатовъ Торговый Дом, Челябинск":
//   - "Богатовъ ТД" — объединённая запись ТЕКУЩЕГО (2026) года, план 1 650 000 €.
//     (Сервис=100 000, Оборудование=650 000, Расход.мат.=900 000). ЭТОТ ОСТАВЛЯЕМ.
//   - "ТД Богатов" — отдельная запись ПРОШЛОГО (2025) года, план 586 100 €.
//     (Сервис=75 021, Оборудование=260 000, Расход.мат.=251 080). ИСКЛЮЧАЕМ.
//
// Подстрока "тд богатов" матчит ТОЛЬКО второй лист (lowercase "тд богатов"),
// но НЕ первый ("богатовъ тд" НЕ содержит "тд богатов" как подстроку).
export const EXCLUDED_DEALERS = [
  "тд богатов",
];
```

## Печать карточки дилера

На вкладке «Сводная» у каждой строки таблиц (выполнение и сервис) есть кнопка-иконка принтера. На вкладке «Дилер» такая же кнопка есть в правом верхнем углу. Клик открывает отдельное окно с карточкой дилера в формате, готовом к печати:

- Заголовок: «Условия работы ТПС 2026 ф.г. — Расчёт скидок по итогам N кв.»
- Имя дилера, План 2026, Выручка 2025
- Таблица «Категории — план / факт / выполнение / скидка» (Оборудование / Расход. мат. / Итого / Сервис)
- Описание шкал
- Помесячные отгрузки за финансовый год (Апр → Мар) с итогом
- Автоматический вызов диалога печати браузера
